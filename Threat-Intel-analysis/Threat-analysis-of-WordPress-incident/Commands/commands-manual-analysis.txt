1) View the start of the log (first 10 lines)
   head -n 20 access.log
   -> Quick check of format: IP, timestamp, request, status, user-agent.

2) View the end of the log (last 10 lines)
   tail -n 20 access.log
   -> See latest timestamps; check if the date you investigate exists.

3) Extract client IPs (first field)
   cat access.log | cut -d ' ' -f 1
   -> The first field in common Apache logs is the client IP.

4) Count requests per IP (most active first)
   cat access.log | cut -d ' ' -f 1 | sort | uniq -c | sort -nr
   -> Find noisy or suspicious IPs.

5) Save IP frequency to file
   cat access.log | cut -d ' ' -f 1 | sort | uniq -c | sort -nr > data/processed/IPs.txt

6) Extract user-agents (field inside quotes)
   cat access.log | cut -d '"' -f 6
   -> User-agent often in the 6th quoted field; experiment if format differs.

7) Clean and count user-agents
   cat access.log | cut -d '"' -f 6 | sort | uniq -c | sort -nr > data/processed/useragents.txt

8) Look for POST requests (often uploads / logins)
   grep 'POST' access.log

9) Exclude 403s (forbidden)
   grep 'POST' access.log | grep -v ' 403 '

10) Get IPs for POST requests (clean list)
    grep 'POST' access.log | grep -v ' 403 ' | cut -d ' ' -f 1 | sort | uniq -c | sort -nr > data/processed/POST_IPs.txt

11) Search for suspicious filenames (example: fr34k)
    grep 'fr34k' access.log

12) Search for plugin paths (case-insensitive)
    grep -i 'contact-form-7' access.log
    grep -i 'simple-file-list' access.log

13) Build simple hourly timeline
    cut -d'[' -f2 access.log | cut -d':' -f1-2 | sort | uniq -c | sort -nr > data/processed/hourly_timeline.txt

14) Notes on OSINT checks:
    - Use AbuseIPDB and VirusTotal to look up the most active IPs.
    - Do not paste raw log content into public sites (sanitize if needed).
